CollectionReference Overview

Represents a Cloud Firestore Collection.

Collections are implicit namespaces for Firestore Documents. They are created when the first document is inserted, and cease to exist when the last document is removed.

Extends Google\Cloud\Firestore\Query

Example

use Google\Cloud\Firestore\FirestoreClient;

$firestore = new FirestoreClient();
$collection = $firestore->collection('users');
Available methods: __construct, add, document, documents, endAt, endBefore, id, limit, limitToLast, listDocuments, name, newDocument, offset, orderBy, parent, path, select, startAfter, startAt, where

Parameters

A Firestore Value Mapper.

# add

Generate a new document reference, and insert it with the given field data.

This method will generate a random document name. If you wish to create a document with a specified name, create a reference with Google\Cloud\Firestore\CollectionReference::document(), then call Google\Cloud\Firestore\DocumentReference::create() to insert the document.

This method immediately inserts the document. If you wish for lazy creation of a Document instance, refer to Google\Cloud\Firestore\CollectionReference::document() or Google\Cloud\Firestore\CollectionReference::newDocument().

Parameters

fieldsarray

An array containing field names paired with their value. Accepts a nested array, or a simple array of field paths.

optionsarray

Configuration Options.

Returns

Google\Cloud\Firestore\DocumentReference

Example

$newUser = $collection->add([
    'name' => 'Kate'
]);

# endAt

An end point for the query results.

Ends results at the provided set of field values relative to the order of the query. The order of the provided values must match the order of the order by clauses of the query. Values in the given cursor will not be included in the result set.

Multiple invocations of this call overwrite previous calls. Calling endBefore() will overwrite both previous endBefore() and endAt() calls.

Implemented in Google\Cloud\Firestore\Query

Parameters

fieldValuesarray, Google\Cloud\Firestore\DocumentSnapshot

A list of values, or an instance of Google\Cloud\Firestore\DocumentSnapshot defining the query end point.

Returns

Google\Cloud\Firestore\Query

A new instance of Query with the given changes applied.

Example

$query = $query->orderBy('age', 'ASC')->endAt([17]);
$usersYoungerThan18 = $query->documents();

# endBefore

An end point for the query results.

Ends results before the provided set of field values relative to the order of the query. The order of the provided values must match the order of the order by clauses of the query. Values in the given cursor will be included in the result set, if found.

Multiple invocations of this call overwrite previous calls. Calling endBefore() will overwrite both previous endBefore() and endAt() calls.

Implemented in Google\Cloud\Firestore\Query

Parameters

fieldValuesarray, Google\Cloud\Firestore\DocumentSnapshot

A list of values, or an instance of Google\Cloud\Firestore\DocumentSnapshot defining the query end point.

Returns

Google\Cloud\Firestore\Query

A new instance of Query with the given changes applied.

Example

$query = $query->orderBy('age', 'ASC')->endBefore([18]);
$usersYoungerThan18 = $query->documents();

# name

Get the collection name.

Names are absolute. The result of this call would be of the form projects/<project-id>/databases/<database-id>/documents/<relative-path>.

# startAfter

A starting point for the query results.

Starts results after the provided set of field values relative to the order of the query. The order of the provided values must match the order of the order by clauses of the query. Values in the given cursor will not be included in the result set.

Multiple invocations of this call overwrite previous calls. Calling startAt() will overwrite both previous startAt() and startAfter() calls.

Implemented in Google\Cloud\Firestore\Query

Parameters

valuemixed

Returns

Google\Cloud\Firestore\Query

A new instance of Query with the given changes applied.

Example

$query = $query->where('firstName', '=', 'John');

Filtering against null and NAN is supported only with the equality operator.

$query = $query->where('coolnessPercentage', '=', NAN);

Use array-contains to select documents where the array contains given elements.

$query = $query->where('friends', 'array-contains', ['Steve', 'Sarah']);
